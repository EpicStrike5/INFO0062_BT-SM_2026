Two optional flags can be combined in any order:

- `--display` â€” opens a Swing window showing the solved puzzle with jigsaw-shaped pieces drawn using cubic BĂ©zier curves. If no solution exists, it shows the deepest valid partial state found within 3 seconds.
- `--timer` â€” prints the solving time to stderr.

## Solver design

**Feasibility check.** Before any search, the program counts flat sides, bumps, and pits across all pieces and immediately rejects the puzzle if the flat side count doesn't match the grid perimeter, bumps and pits are unequal, or piece-type counts don't fit the grid dimensions. This catches most unsolvable puzzles instantly without backtracking.

**Phase 1 â€” Border.** Border cells are filled first in clockwise order. Corners only accept `CornerPiece` instances; edges only accept `EdgePiece` instances. Because outer sides must be flat, each border cell has far fewer valid placements than an interior cell, so placing them first tightly constrains the search space before the interior is touched.

**Phase 2 â€” Interior.** Interior cells are filled left-to-right, top-to-bottom using only `InsidePiece` instances. By the time the solver reaches any interior cell, its top and left neighbours are already placed, so two of its four sides are already determined.

**Why it handles up to ~15Ă—15 well but struggles on 50Ă—50.** The two phases each use plain backtracking with no look-ahead. On smaller grids the search tree is pruned aggressively enough by neighbour constraints that the solver rarely has to backtrack far. On a 50Ă—50 grid (2304 interior cells), a wrong choice made early in the interior only becomes a dead end hundreds of cells later â€” by then the solver has to unwind a long chain of placements before it can try something different, leading to tens of millions of backtracks without making net progress.

## Class structure

| Class | Role |
|---|---|
| `Element` | Abstract base for all pieces. Holds the four sides, provides `rotate()` and `isCompatibleWith()`. The factory `Element.of()` picks the correct subtype by counting flat sides. |
| `CornerPiece` / `EdgePiece` / `InsidePiece` | Subtypes used as type filters during the search â€” corners only go in corners, edges only on the border. |
| `Placement` | Immutable record pairing a piece index with a rotation. Stored in the solution grid. |
| `Puzzle` | Data container (grid, piece list, used-flags) and program entry point. |
| `PuzzleSolver` | Backtracking engine: feasibility check, border fill, interior fill, and partial-solve for display mode. |
| `PuzzleIO` | Reads `.pzl` files and generates random puzzles. |
| `PuzzleError` | All error messages in one place, printed to stderr. |
| `PuzzleDisplay` | Swing window with antialiased jigsaw rendering. Only used when `--display` is passed. |
